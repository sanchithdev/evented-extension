Privacy Policy

Effective Date: August 6, 2025

Data Handled

  • User Input: Only the text you highlight or type is used to create events. It is sent to OpenAI only when you click "Create Event."
  • API Key: Your personal OpenAI API key is stored only in your browser using Chrome’s storage.local. It is never sent anywhere else.
  • Model Selection: Your chosen default AI model is also stored locally for convenience and not shared externally.

How Data Is Stored

All settings and preferences are saved in Chrome’s storage.local. This means everything is stored only on your device and never transmitted to any Evented-controlled backend. Nothing is synced to the cloud or shared with us.

When Data Is Sent

When you click “Create Event,” the highlighted text is sent over a secure HTTPS connection to OpenAI for processing. If you have enabled Google Calendar access, Evented will create an event using Google’s Calendar API.

Third-Party APIs

  • OpenAI API – used to interpret your text into event details using the model you selected.
  • Google Calendar API – used to create events on your behalf, only if you've authenticated with Google.

These services process your data as part of the functionality you choose to use. Please review their respective privacy policies to understand how they manage your data.

Data Retention

Evented does not retain any input data on our side. The data sent to OpenAI is discarded after processing. Your API key and model choice stay locally stored until you delete or update them.

User Control

You can update or remove your OpenAI API key and model settings at any time through the extension. You can also revoke Google Calendar access via your Google account settings or by signing out.

Changes to This Policy

If changes are made to this policy, they will be reflected here. It is recommend to check this page occasionally, especially if new features are introduced.

By using Evented, you agree to and accept this privacy policy.
