Privacy Policy

Effective Date: August 11, 2025

Data Collected and Handled

  • User Text Input: Only the text you highlight or type, and then submit by clicking the create button, is used to create events.
  • Google User Data: If you sign in with Google, Evented uses only an authentication token and permission to insert new events into your primary calendar. This data is used solely to create events you request. It does not read or modify any existing events, nor access other calendar data.
  • API Key: Your personal OpenAI API key that is used to interpret your highlighted or typed text into structured calendar event details (title, location, start/end time, etc.).
  • Model Selection: Your chosen OpenAI model to process your input text into structured calendar event details.

Data Storage and Retention

All settings, API keys, and model preferences are saved locally using Chrome’s storage.local until you remove (reset) them. No data is stored on Evented-controlled servers. Google authentication tokens are stored securely using Chrome Identity API and can be revoked at any time by signing out. Input event data is held only temporarily in memory while processing your request, then discarded.

When and How Data Is Sent & Used

  • OpenAI API: When you click the “Create” button, the highlighted or typed text is sent over a secure HTTPS connection to convert it into structured event details (title, location, start_time, end_time).
  • Google Calendar API: If signed in and access is granted, Evented sends the event details to your Google account’s primary calendar in standard Google Calendar format. No other calendar data is accessed or read.
  • Signed Out: If you are signed out, no Google Calendar API calls are made and no Google user data is accessed.

Data Sharing and Disclosure

Evented does not sell, rent, or share your personal data with any third party for purposes other than processing your text input to create calendar events. Data is shared only with OpenAI (to interpret and structure your text input) and Google (to create Google Calendar events).

Both OpenAI and Google handle your data under their respective privacy policies: OpenAI Privacy Policy and Google Privacy Policy.

Data Protection

  • All communication with external APIs uses secure HTTPS encryption.
  • OAuth 2.0 is used for Google authentication, ensuring tokens are securely handled.
  • Your API key is stored locally in Chrome and never transmitted to any external Evented-controlled server.

User Control

You may:

  • Remove or update (reset) your OpenAI API key and preferences from within the extension.
  • Sign out of Google from the extension at any time to revoke all Google Calendar access.

Changes to This Policy

If changes are made to this policy, the “Effective Date” will be updated, and the revised version will be posted here. Please check this page periodically for updates.

By using Evented, you agree to and accept this privacy policy.
